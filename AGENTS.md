# AGENTS.md

## 项目基准

- 项目名：导题斩题小工具。
- 平台：微信小程序。
- 需求源：优先阅读并遵守 `导题斩题小工具-PRD.md`。
- 目标用户：福建省职教高考考生。
- MVP 学科：数学、英语、政治。
- 核心机制：掌握知识点后触发“斩题”，被斩知识点暂不重复出现，30 天后低频复活。

## 产品范围

- 保留 PRD 中的 MVP 范围：刷题、斩题、错题、打卡、题库导入、资讯、历届真题、微信云开发 CMS。
- MVP 不实现完整成就系统，只保留轻量完成感反馈。
- 免费功能优先，广告只能作为弱打扰变现，不得遮挡答题、解析、复习和导入流程。

## 设计原则

- 整个 App 以蓝色为主色。
- 设计风格严格遵循 iOS26 液态玻璃方向：半透明、蓝色折射感、边缘高光、轻模糊、浮动层级、细描边、柔和动态。
- 禁止沿用传统“大白卡片 + 普通阴影”作为主视觉。
- 禁止大面积使用朱红、金色、橙色、紫色。红色只用于错误/危险反馈，绿色只用于正确反馈。
- 结构图标不得使用 emoji，统一使用线性矢量图标。
- 所有触控目标不小于 44pt，并适配微信小程序安全区。

## 视觉 Token

- 主蓝：`#007AFF`。
- 深蓝文本：`#102A56`。
- 背景浅蓝：`#F3F8FF`。
- 玻璃表面：浅色模式使用 `rgba(255,255,255,0.58)`，深色模式使用 `rgba(15,32,64,0.52)`。
- 玻璃描边：`rgba(255,255,255,0.45)` 与 `rgba(0,122,255,0.20)`。
- 高光蓝：`#5AC8FA`。
- 正确反馈：`#34C759`。
- 错误反馈：`#FF3B30`。

## 交互与动效

- 斩题动画使用蓝色液态光晕、玻璃形变、轻粒子和印章反馈。
- 斩题动画必须可跳过，默认总时长不超过 2.5 秒。
- 开启减少动态效果时，斩题动画降级为短淡入淡出和静态完成标识。
- 答题正确反馈以清晰结果和解析为主，动效克制。
- 答题错误不得强闪烁，不得造成阅读干扰。
- 页面转场遵循小程序和 iOS 常见导航习惯，返回路径必须清晰。

## 核心业务规则

- 官方题库按 `knowledgePointId` 斩题。
- 自导入题库按 `questionId` 斩题。
- 斩题触发需满足最近 10 题正确率不低于 80%，样本不足 10 题不得触发。
- 斩题需二次确认，并保留每日 3 次回滚机会。
- 错题复习遵循 PRD 中的 SM-2 变体间隔序列。
- 选题算法中“距离上次出现越久权重越高”必须与公式一致，不得使用递减后仍声称递增的表达。

## 数据与合规

- 新增功能时优先补齐数据来源、状态字段、失败状态和回滚路径。
- 题库导入必须提供预览、修正、失败提示和导入进度。
- OCR、相机、文件上传、订阅消息、题库分享必须有明确授权与降级路径。
- 真题、资讯和用户上传内容需考虑版权、举报和审核。
- 用户数据需支持缓存清理、同步冲突处理和异常恢复。

## 验收要求

- 小屏、大屏、横屏、深色模式、浅色模式均不能出现文字溢出或按钮重叠。
- 长题干、多选题、填空题、简答题、解析内容必须可读。
- 离线、弱网、OCR 超限、题目删除、答案争议、存储满都要有明确反馈。
- 所有页面保持蓝色液态玻璃一致性，不得出现风格断层。
