## 二、P0 严重问题（审核阻断 / 功能故障）

### P0-1. 缺少 `requiredPrivateInfos` 隐私接口声明 ❌ 审核必拒

- **文件**：`miniprogram/app.json`
- **问题**：代码中调用了 `wx.chooseMedia`、`wx.chooseMessageFile` 等隐私 API，但 `app.json` 中未声明 `requiredPrivateInfos`。2023 年 9 月微信新规后，调用这些接口必须预先声明，否则审核被拒且调用静默失败。
- **修复**：
```json
"requiredPrivateInfos": ["chooseMedia", "chooseMessageFile"]
```

### P0-2. 隐私协议入口缺失 ❌ 审核必拒

- **文件**：`miniprogram/app.json`、`miniprogram/pages/`
- **问题**：`__usePrivacyCheck__: true` 已开启，但全项目无隐私政策页面，无隐私弹窗跳转入口。审核会判定"未提供隐私政策查看入口"。
- **修复**：新增 `pages/privacy/index` 隐私政策页面，在"我的"页提供入口。

### P0-3. 占位页面未完成功能 ❌ 审核必拒

- **文件**：`miniprogram/pages/placeholder/index.wxml`、`pages/index/index.js:248-253`
- **问题**：页面仅显示"页面框架已接入，下一步会补齐数据"。首页的"政策资讯"和"在线答疑"均跳转至此占位页。微信审核以"页面功能不完整"拒绝。
- **修复**：上线前移除 `policy`、`qa` 两个 quickAction 入口，或接入真实内容。

### P0-4. 生产代码残留 mockData 测试数据 ❌ 审核必拒

- **文件**：`miniprogram/pages/practice/index.js:1,487,495`
- **问题**：`switchToNextClass` 完全依赖 `mockData` 切换题类，属于测试数据残留 + 真实功能未接入。
- **修复**：移除 mockData 引用，从云端或正式题库数据源加载。

### P0-5. 深色模式下文字几乎不可见 ❌ 全局可用性

- **文件**：13+ 个 WXSS 文件，30+ 处
- **问题**：`.theme-dark` 下将次要文字颜色设为 `#1A2538`（深蓝黑），而深色模式背景为 `#0F172A`（深蓝黑），**文字与背景几乎同色**。影响所有分包页面的副标题、卡片描述、统计标签、解析正文等核心信息。
- **涉及文件**：`index.wxss:966`、`mine/index.wxss:395`、`practice/index.wxss:765,792,806`、`bank/list/index.wxss:443`、`wrong/list/index.wxss:421`、`wrong/review/index.wxss:661`、`feedback/index.wxss:124`、`memorize/index.wxss:332`、`history/index.wxss:276`、`exam/list/index.wxss:241`、`placeholder/index.wxss:86` 等。
- **修复**：全局将 `.theme-dark` 下的 `color: #1A2538` 替换为 `rgba(255,255,255,0.6)` 或 `#94A3B8`。

### P0-6. 云函数 `deleteBank` 越权漏洞 ❌ 安全严重

- **文件**：`cloudfunctions/quickstartFunctions/index.js`（deleteBank 类型处理）
- **问题**：`ownerId` 为空字符串时条件短路，导致任何用户可删除官方题库。一行修复即可避免灾难性数据丢失。
- **修复**：在删除前校验 `if (!ownerId) return { error: '无权操作' };`

### P0-7. 管理端 API 完全无 RBAC ❌ 安全严重

- **文件**：`cloudfunctions/admin-api/index.js`
- **问题**：管理端 API 没有任何身份验证/权限校验，任意用户可执行删除题库、修改题目等高危操作。
- **修复**：增加管理员 openid 白名单校验，所有写操作前验证调用者身份。

### P0-8. 邮件正文 XSS 注入漏洞 ❌ 安全严重

- **文件**：`cloudfunctions/quickstartFunctions/index.js`（nodemailer 相关）
- **问题**：邮件正文直接拼接用户输入，存在 XSS 风险。
- **修复**：对用户输入做 HTML escape 处理后再拼接。

### P0-9. 分享码永久有效无过期校验 ❌ 安全

- **文件**：`cloudfunctions/quickstartFunctions/index.js`（getSharedBank 类型）
- **问题**：`getSharedBank` 未校验 `expireAt`，分享码永久有效，存在数据泄露风险。
- **修复**：查询时增加 `if (shareData.expireAt && Date.now() > shareData.expireAt) return { error: '分享已过期' };`

### P0-10. 错题本数据完全未云同步 ❌ 数据丢失

- **文件**：`miniprogram/utils/cloudSync.js:118-141, 147-172`
- **问题**：`collectLocalData()`、`applyCloudToLocal()`、`saveSection()` 均未处理 `wrongBook` 和 `errorTracking`。用户换设备/重装后错题本全部丢失，SM-2 复习进度归零。
- **修复**：在 cloudSync 中补充 wrongBook 和 errorTracking 的同步逻辑。

### P0-11. 分享接收页数据不持久化 ❌ 数据丢失

- **文件**：`miniprogram/pages/bank/share-receive/index.js:138-139`
- **问题**：直接 `mockData.banks.push(newBank)`，mockData 是内存对象，冷启动后丢失。用户导入分享题库后下次打开消失。
- **修复**：通过云函数写入云数据库，或使用 `wx.setStorageSync` 持久化。

### P0-12. 错题复习 SM-2 重复更新 ❌ 数据错误

- **文件**：`miniprogram/pages/wrong/review/index.js:334-377, 380-416`
- **问题**：`nextQuestion` 和 `finishReview` 对同一道题都调用了 `wrongBook.reviewAnswer`，SM-2 间隔被推进两次，导致复习间隔异常加长。
- **修复**：增加 `_reviewed` 标记防止重复处理。

### P0-13. 背题页"没记住"按钮缺失 ❌ 功能缺失

- **文件**：`miniprogram/pages/memorize/index.wxml`、`index.js:192-203`
- **问题**：JS 定义了 `markNotMemorized` 方法但 WXML 无对应按钮。用户无法标记未掌握题目，统计数据失真。
- **修复**：在 `.memorize-actions` 区域增加"没记住"按钮。

### P0-14. 选题算法未实现权重 ❌ 核心业务缺失

- **文件**：`miniprogram/pages/practice/index.js:480-484`
- **问题**：PRD 要求"距离上次出现越久权重越高"，实际代码是纯随机 `Math.floor(Math.random() * candidates.length)`，完全未实现权重机制。
- **修复**：实现加权随机算法，基于每题/题类上次出现时间计算权重。

### P0-15. 云端题库查询丢失筛选条件 ❌ 数据错乱

- **文件**：`miniprogram/utils/questionData.js:84-96`
- **问题**：构建了 `condition` 含 `knowledgePointId`/`questionClassId`，但云函数调用 `data: { type: 'getBankQuestions', bankId }` 完全未传入 condition，导致返回题库全部题目，斩题进度追踪错乱。
- **修复**：将 condition 参数传入云函数调用。

### P0-16. 打卡连续天数计算时区 BUG ❌ 数据错误

- **文件**：`miniprogram/utils/checkinManager.js:82`
- **问题**：`toISOString().split('T')[0]` 使用 UTC 时间。东八区凌晨 00:00-08:00 打卡时返回前一天日期，连续打卡判断错位。
- **修复**：使用本地日期构造函数。

### P0-17. 隐私授权检查失败时降级放行 ❌ 合规风险

- **文件**：`miniprogram/utils/privacyAuth.js:73-78`
- **问题**：`wx.getPrivacySetting` 调用失败时直接 `resolve()` 放行，注释写"避免阻断核心功能"。隐私合规要求是强制的，审核会被拒。
- **修复**：失败时应 `reject`，由调用方决定降级策略。

### P0-18. 30+ 处 emoji 用作结构图标 ❌ 设计规范硬性违规

- **文件**：`index.wxml:35,119`、`practice/index.wxml:165`、`practice/result.wxml:8,69,73`、`wrong/list/index.wxml:116`、`wrong/review/index.wxml:8,186`、`history/index.wxml:29`、`exam/list/index.wxml:55`、`slash/manage/index.wxml:73`、`bank/list/index.wxml:136`、`bank/import/index.wxml:38,45,50,75,82,87,113,178,182,222,223`、`bank/import-preview/index.wxml:264`、`bank/share-receive/index.wxml:14,22,41`
- **问题**：PRD 明确"结构图标不得使用 emoji，统一使用线性矢量图标"，但 30+ 处违反。`images/kzg/` 已有完整 SVG 图标库可复用。
- **修复**：用 `/images/kzg/` 下 SVG 替换全部 emoji。

### P0-19. 错误色与 PRD 不符 + 大面积橙色/朱红 ❌ 设计规范违规

- **文件**：多处 WXSS
- **问题**：
  - 自定义错误色 `#F2545B`/`#FF6B4A`/`#FF5C4A`/`#EF4444`，与 PRD 规定 `#FF3B30` 全部不符
  - `practice/index.wxss:699` 斩题浮动按钮用 `rgba(249,115,22,0.82)` 大面积橙色
  - 多页朱红渐变按钮 `linear-gradient(135deg, #FF6B4A, #FF5C4A)`
  - `slash/manage/index.wxss:153` 金色 `#F5A623`/`#D97706`
  - `index/index.wxss:173` 金色渐变 `rgba(245,166,35,...)`
- **修复**：全局替换错误色为 `#FF3B30`；斩题按钮改蓝色液态光晕；金色改 `#5AC8FA`。

### P0-20. "我的"页使用紫色 ❌ 设计规范违规

- **文件**：`pages/mine/index.wxss:34,136,319`
- **问题**：`.ambient-strip` 紫色光晕 `rgba(124,92,252,0.08)`、`.purple-bg` 紫色背景、`.dock-label` 紫蓝渐变 `#7C5CFC`。
- **修复**：紫色改为高光蓝 `#5AC8FA` / `rgba(90,200,250,...)`。

### P0-21. 触控目标低于 44pt ❌ 可用性

- **文件**：多处 WXSS
- **问题**：`option-item` 76rpx、`judge-btn` 72rpx、`bank-delete` 44rpx、`clean-btn` 76rpx、`dock-item` 56rpx、`tf-item` 56rpx、`q-image-delete` 56rpx 等，均低于 88rpx 下限。
- **修复**：所有可点击元素 `min-height` ≥ 88rpx。

### P0-22. import-preview 输入即全量 setData ❌ 性能严重

- **文件**：`miniprogram/pages/bank/import-preview/index.js:169-218`
- **问题**：每次输入字符都 `this.setData({ questions })`，questions 是完整题目数组（最多 500 题），500 题规模下明显卡顿。
- **修复**：使用路径表达式 `this.setData({ [`questions[${index}].${field}`]: value })`。

### P0-23. project.config.json 残留无效调试入口

- **文件**：`project.config.json:64-72`
- **问题**：`condition.miniprogram.list` 引用不存在的 `pages/databaseGuide/databaseGuide`，属模板残留。
- **修复**：清空为 `"list": []`。

### P0-24. 斩题动画无法跳过且未适配 reduce-motion

- **文件**：`pages/practice/index.wxss:723`、全项目
- **问题**：PRD 要求"斩题动画可跳过，≤2.5秒"。实际 `.slash-float-ring` 动画 `infinite` 无限循环，无跳过按钮，全项目无 `prefers-reduced-motion` 适配。
- **修复**：动画改为有限次数，增加跳过按钮，全局加 reduce-motion 媒体查询。

### P0-25. 斩题管理页深色模式机制不一致

- **文件**：`pages/slash/manage/index.wxss:213-248`
- **问题**：唯一使用 `@media (prefers-color-scheme: dark)` 而非 `.theme-dark` 类，用户在 App 内手动切换主题时此页面不响应。
- **修复**：改为 `.theme-dark` 类选择器，与其他页面统一。
