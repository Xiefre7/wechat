# 2026-07-02 工作记录

## 启动流程性能优化

针对微信小程序冷启动流程进行了以下优化（严格不破坏功能/UI/卡片设计）：

### 1. 主包体积瘦身（1.7MB → 1.3MB，减少约 420KB）
- 删除未引用的 `images/bg/` 目录（与 `bg-light/` 重复）
- 删除 11 张未使用的云开发教程截图（ai_example、cloud_dev、database 等）
- 删除未使用的 `images/avatar.png`、`images/arrow.svg`、`images/copy.svg`
- 删除未使用的 `images/icons/` 下 16 个 png/svg 图标（仅保留 share.svg）
- 删除未使用的 `images/tab/` 目录

### 2. app.js onLaunch 去阻塞
- `getSystemInfoSync()` → `getAppBaseInfo()`（轻量级 API，带降级保护）
- `authManager.silentInit()` 和 `slashManager.checkAndRevive()` 延迟到 `setTimeout(0)` 执行
  - 两者均不影响首屏展示，延迟执行可加速冷启动
  - silentInit 是异步云调用，checkAndRevive 是同步 storage 读取+遍历

### 3. 首页 onLoad setData 合并（5 次 → 1 次）
- `pages/index/index.js`: updateGreeting + loadUserData + loadCheckinData + loadStudyData + loadHistoryData 合并为单次 setData
- `pages/mine/index.js`: 4 次独立 setData 合并为 1 次
- 各 load 方法保留供 onShow 调用，不影响运行时刷新逻辑

### 4. 初始渲染缓存
- `pages/index/index.json` 和 `pages/mine/index.json` 添加 `"initialRenderingCache": "static"`
- 使视图层无需等待逻辑层初始化即可展示缓存的初始渲染结果

### 已有优化（确认无需修改）
- `lazyCodeLoading: "requiredComponents"` 已启用
- `preloadRule` 已配置（首页预加载 bank/practice 分包）
- `slashManager` 使用 cloudSync 懒加载模式
- 分包结构合理（8 个子分包）

## 骨架屏优化

为首页（`pages/index/index`）添加骨架屏，消除冷启动白屏空档期。

### 实现方案
- **index.js**: `data` 中添加 `loading: true`（在 initDataSendTime 时发送到视图层），`onLoad` 的 setData 中添加 `loading: false`
- **index.wxml**: scroll-view 内用 `wx:if="{{loading}}"` 控制骨架屏显示，`wx:else` 包裹真实内容
- **index.wxss**: 追加 `sk-` 前缀骨架屏样式，尺寸精确匹配真实组件

### 骨架屏覆盖区域
1. hero 区（头像圆 + 问候语文字条）
2. 统计条（3 个数字+标签占位 + 分隔线）
3. 两张主卡片（图标圆 + 标题 + 描述 + 双指标 + 箭头按钮）
4. 快捷操作（2 个玻璃按钮占位）
5. 历史记录（标题行 + 2 张历史卡片占位）

### 设计要点
- 复用 `glass-card` 类保持液态玻璃一致性
- shimmer 动画使用蓝色折射光带（`rgba(74,143,234,...)` 渐变扫过）
- 深色模式适配（白色半透明渐变）
- `prefers-reduced-motion` 降级为静态色块
- `loading: true` 写在 data 中确保 initDataSendTime 即可渲染
- `onShow` 不触碰 loading 变量，热启动不闪骨架屏
