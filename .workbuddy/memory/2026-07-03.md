## 修复登录页隐私协议勾选 Bug
- 文件：miniprogram/pages/login/index.js
- 根因：`_checkCanLogin()` 在 `setData` 对象字面量中被调用，JS 求值顺序导致它读取 `this.data` 旧值，`canLogin` 状态始终滞后一步
- 表现：用户勾选→取消勾选隐私协议后，`canLogin` 错误为 true，导致未勾选也能登录
- 修复：三个事件处理函数（onChooseAvatar/onNicknameInput/onPrivacyChange）改用新值内联计算 `canLogin`，不再调用 `_checkCanLogin()` 读取旧 data
- 额外安全网：`handleLogin` 入口处直接校验 `agreedPrivacy`，防止任何边界情况绕过

## 修复退出登录后导航回 mine 页面昵称头像残留 Bug
- 文件：miniprogram/pages/mine/index.js
- 根因：模块级变量 `_userInfo` 在 JS 模块首次加载时求值，之后被缓存。退出登录后 `app.globalData.userInfo` 已重置为默认值，但 `_userInfo` 仍持有旧数据
- 触发路径：退出登录（mine 页 setData 修复了）→ 点学习（redirectTo 销毁 mine 页）→ 点我的（redirectTo 重建 mine 页）→ `data` 初始化用过期的 `_userInfo` → 昵称头像又出现
- 加剧因素：`onLoad` 不刷新用户信息；`onShow` 首次跳过且不含 avatarUrl/nickname 字段
- 修复：
  1. `onLoad` 中从 `app.getUserInfo()` 读取最新值并 setData（在首次渲染前执行，无闪烁）
  2. `onShow` 的 setData 中补充 `avatarUrl` 和 `nickname` 字段（后续 onShow 也刷新）
- 对比：index 页面无此问题，因为其 `data` 初始化为空字符串，`onLoad` 从 `app.getUserInfo()` 设置

## 优化 Word 文档数学公式识别能力
- 根因：`mammoth.extractRawText()` 完全丢弃 OMML（Office MathML）公式，Word 公式编辑器创建的数学内容全部丢失
- 解决方案：新增 `lib/docxMathExtractor.js` 模块，直接解压 .docx（ZIP）→ 解析 `word/document.xml` → 按文档顺序提取 `<w:t>` 普通文本 + `<m:oMath>` 数学公式
- OMML → 可读文本转换（递归解析器）：
  - `m:f` 分数 → 分子/分母（复杂内容自动加括号）
  - `m:sSup` 上标 → 优先 Unicode 上标（x²），多字符用 `^(content)`
  - `m:sSub` 下标 → 优先 Unicode 下标（x₁），多字符用 `_(content)`
  - `m:rad` 根号 → √(被开方数)
  - `m:nary` N元运算 → ∑/∫/∏ + 上下限
  - `m:d` 定界符 → (内容)/[内容]/{内容}
  - `m:acc` 重音 → x̄/x̂/x⃗（组合字符）
  - `m:m` 矩阵 → [a, b; c, d]
  - `m:eqArr` 方程组 → 逐行换行
  - `m:func` 函数 → sin(x)/log(x)
- LaTeX 残留清理（`normalizeMathSymbols`）：`\frac{}{}`→分式、`\sqrt{}`→√、`\sum`→∑、`\int`→∫、`\pi`→π、`\alpha`→α、`\leq`→≤、`\times`→× 等 80+ 符号映射
- 解析策略：docxMathExtractor 优先 → mammoth 回退（双保险，不破坏旧功能）
- 新增依赖：`adm-zip@^0.5.10`（纯 JS ZIP 解压）
- 文件：lib/docxMathExtractor.js（新建）、lib/optionNormalizer.js（增强）、index.js（parseWordDocument 改造 + parseOcrText 输出归一化）
