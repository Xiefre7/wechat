## 一、配置文件检查

### 🔴 P0-1：缺少隐私保护指引配置（合规阻断）

**文件**：`miniprogram/app.json`

`app.json` 中**完全缺失**以下微信隐私合规必需配置：
- `__usePrivacyCheck__`：未开启隐私合规检查
- `requiredPrivateInfos`：未声明项目实际使用的隐私接口

项目代码中使用了 `wx.chooseMedia`、`wx.chooseMessageFile` 等隐私 API（见 `utils/imageUploader.js:80`、`pages/bank/import/index.js:147/203/357/368`、`pages/example/index.js:420`），但未做任何声明。

**影响**：微信自 2023-09 起强制要求，缺失将导致：
1. 隐私 API 调用直接失败
2. 提审被拒
3. 已上架版本可能被下架

**整改**：在 `app.json` 增加：
```json
"__usePrivacyCheck__": true,
"requiredPrivateInfos": ["chooseMedia", "chooseMessageFile"]
```
并配置用户隐私保护指引（在小程序后台「设置→用户隐私保护指引」中填写）。

### 🔴 P0-2：未配置系统级深色模式（与已实现的深色逻辑脱节）

**文件**：`miniprogram/app.json`、缺失 `theme.json`

- `app.json` 未配置 `"darkmode": true`
- 不存在 `theme.json` 文件
- 未使用 `wx.onThemeChange` 监听系统主题变化（“跟随系统”模式下切换系统主题不会实时响应）

**整改**：在 `app.json` 增加 `"darkmode": true`，创建 `theme.json` 定义 WXSS 变量映射；或在 `app.js` 的 `onLaunch` 中注册 `wx.onThemeChange` 回调以实现系统主题实时跟随。

## 三、云函数审查

### `cloudfunctions/quickstartFunctions/`

#### 🔴 P0-4：示例 CRUD 函数无身份验证（安全隐患）

**文件**：`index.js:110-185`

`selectRecord`、`updateRecord`、`insertRecord`、`deleteRecord` 四个函数操作 `sales` 集合，但**无任何身份校验**，且通过 `exports.main` 的 switch 对外暴露。任何小程序用户均可直接调用进行增删改查。

```javascript
const deleteRecord = async (event) => {
  await db.collection("sales").where({ _id: event.data._id }).remove(); // 无鉴权
};
```

**整改**：从 `exports.main` 的 switch 中移除这四个 case（属云开发模板示例代码），或加入管理员鉴权。

#### 🔴 P0-5：shareBank 函数未验证用户身份

**文件**：`index.js:1122-1145`

```javascript
const shareBank = async (event) => {
  const { bank, questions } = event;
  // 未获取 cloud.getWXContext().OPENID
  await db.collection('sharedBanks').add({ data: { shareCode, bank, questions, ... } });
};
```

任何调用者均可向 `sharedBanks` 集合写入任意数据，可被滥用造成数据库膨胀或存储型 XSS（通过分享的题目内容）。

**整改**：获取 openid 记录分享者身份，并校验 `bank`/`questions` 的合法性。

#### 🟠 P1-5：单云函数承载 20+ 功能，体积过大

**文件**：`index.js`（1358 行）

`quickstartFunctions` 通过 `event.type` 分发承载了：登录、二维码、题库导入/删除、Excel/Word/OCR 解析、反馈邮件、错题本、题库分享、用户资料、真题查询等 20+ 功能。

**影响**：冷启动慢、单点故障、维护困难、依赖臃肿（xlsx + mammoth + nodemailer 全部打进一个函数）。

**整改**：按业务域拆分为独立云函数（如 `question-import`、`feedback`、`wrong-book`、`user-service`、`exam-service`）。

#### 🟠 P1-6：importBank 逐条写入，性能瓶颈 ✅ 已修复（2026-07-02）

**文件**：index.js:254-257

## 四、安全合规检查

### 🔴 P0-8：管理端 Word 文档 HTML 直接注入（存储型 XSS）

**文件**：`管理端网页/js/app.js:1185`

```javascript
previewEl.innerHTML = data.htmlContent || '';  // Word 解析的 HTML 未消毒直接注入
```

管理员上传的 Word 文档经 mammoth 解析为 HTML 后，**未经任何消毒**直接 `innerHTML` 注入。若文档含恶意 `<script>` 或 `<img onerror="...">`，将在管理员浏览器执行，且注入内容随后发布为资讯形成存储型 XSS，可窃取 localStorage 中的管理员 token。

**整改**：引入 DOMPurify 等消毒库对 HTML 进行过滤后再注入。

### 🔴 P0-9：管理端使用 HTTP 明文传输密码

**文件**：`管理端网页/js/api.js:12`

```javascript
const BASE_URL = 'http://localhost:3000'  // HTTP 非 HTTPS
```

管理员登录时用户名密码明文经 HTTP 传输。

**整改**：部署时必须替换为 HTTPS 地址（云函数 HTTP 触发地址）。

### 🟠 P1-7：管理员 Token 存储在 localStorage ✅ 已修复（2026-07-02）

**文件**：`管理端网页/js/auth.js:5-6, 21-22`

`localStorage` 可被 JS 直接读取，配合上述 XSS 漏洞可窃取 token 进行会话劫持。

**整改**：改用 HttpOnly Cookie 存储 token。

### 🟠 P1-8：错误消息未转义直接拼接（潜在 XSS） ✅ 已修复（2026-07-02）

**文件**：`管理端网页/js/app.js` 第 161/233/308/646/1062/1288 行

```javascript
el.innerHTML = '...加载失败：' + res.msg + '...'  // res.msg 未转义
```

**整改**：所有 `res.msg` 拼接处使用已有的 `escHtml()` 函数转义。

### ✅ 安全合规良好实践

- 小程序端未发现任何 `wx.request` 调用外部 HTTP URL，全部通过 `wx.cloud.callFunction` 通信
- `project.config.json` 已启用 `urlCheck: true` 域名白名单检查
