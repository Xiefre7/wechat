# 考斩过 — 微信小程序产品需求文档

## 产品概述

**产品名称**：考斩过
**平台**：微信小程序
**目标用户**：福建省职教高考考生（约6-7万人）
**MVP学科**：数学、英语、政治（公共基础知识）
**商业模式**：全部功能免费 + Banner广告/激励视频广告变现
**核心差异化**："斩题"机制——用户掌握一个知识点后可"斩掉"它，不再重复出现

---

## MVP功能范围

| 功能 | 范围 |
|------|------|
| 刷题 | 官方题库（数英政，5000+题）+ 用户自导入题库 |
| 斩题 | 题类维度触发（做3题正确≥2题即可斩）。官方题库按题类斩，自导入按单题斩 |
| 错题 | 按书籍维度组织。自动收录（官方按知识点错3次，自导入按单题错3次）+ 手动收录 |
| 打卡 | 每日打卡、打卡日历、连续打卡统计 |
| 题库导入 | 拍照OCR + 模板文本粘贴 + Excel上传 + 手动录入，规则引擎解析 + 预览修正 |
| 资讯 | 后台发布 → 前端展示 |
| 历届真题 | 后台发布 → 前端展示 |
| 成就系统 | MVP不做 |
| 管理后台 | 微信云开发CMS，零代码搭建 |

## 关键设计决策

### 1. 斩题机制

**判定方式**：题类维度方案——同一题类做满3题且正确≥2题即可触发斩题。自导入题库仍按单题近10题80%正确率判定。

**适用对象**：
- 官方题库 → 按**题类**斩（知识点的子划分，一个知识点下有多个题类）
- 用户自导入题库 → 按**单题**斩

**斩后衰减**：斩掉的题类/题目7天后自动复活，重新出现在刷题列表中，用户可重新挑战。推送通知提醒"知识已复活，来重新征服它！"

**斩题动画**：
```
阶段1（触发）：累计正确率达标 → 屏幕中央出现金光
阶段2（蓄力）：金光汇聚成剑形，伴随轻微震动
阶段3（斩击）：剑从上向下快速划过屏幕，带刀光轨迹粒子
阶段4（破碎）：知识点名称碎裂散落
阶段5（结算）：碎片汇聚成"已斩"印章盖在知识点上
阶段6（音效）：金属碰撞声 + 胜利号角
总时长约2.5秒，用户可点击跳过
```

**误操作保护**：
- 弹窗二次确认："确认斩掉此知识点？"
- 每日3次回滚机会

### 2. 错题系统

**组织维度**：按**书籍**分类（不论官方还是自导入）

**自动收录规则**：
- 官方题库：同一知识点累计错3次以上 → 该知识点下所有错题加入错题本
- 自导入题库：同一道题错3次以上 → 加入错题本

**手动收录**：答题中点击底部"加入错题本"按钮

**复习机制**：SM-2间隔复习算法
- 间隔序列：1天 → 2天 → 4天 → 7天 → 15天 → 30天 → 60天
- 每次复习答对 → 进入下一间隔
- 答错 → 间隔重置回1天
- 连续3次复习正确 → 从错题本移除（status='mastered'）

**错题统计**：
- 待复习数量提示
- 高频错误知识点排行
- 错题消灭率

### 3. 题库导入（MVP最完善功能）

**导入方式**（按优先级排列）：

| 方式 | 实现 | 用户操作成本 |
|------|------|------------|
| Excel模板上传 | 云函数端SheetJS解析，一键入库 | 低（推荐） |
| 结构化文本粘贴 | 前端规则引擎解析（正则匹配题号/选项/答案） | 中 |
| 手动逐题录入 | 表单填写，逐题提交 | 高 |
| 拍照OCR | 云开发OCR扩展 + 规则引擎解析 + 用户预览修正 | 中 |

**OCR方案**：
- 使用微信云开发OCR扩展（通用印刷体识别，1000次/天免费）
- MVP阶段日均调用预计远低于免费额度，成本≈0
- 设每人每日OCR上限20次防止滥用
- OCR结果 → 规则引擎结构化 → 预览页用户修正 → 确认入库

**结构化处理流程**：
```
上传/拍照 → 原始文本
  → 规则引擎自动识别：题型（单选/多选/判断/填空）、题干、选项（A/B/C/D）、答案、解析
  → 预览页展示解析结果
  → 用户手动修正错误字段
  → 确认入库
```

**限制**：
- 单次导入题目上限：500题
- 文件大小限制：10MB
- 自导入题库无知识点自动分类，按单题管理

**题库分享**：用户可将自导入题库分享给微信好友/群，对方可一键导入

### 4. 留存策略（三层防线）

**第一层：内容供给持续**
- 三科5000+题基础内容池
- 持续新增官方题库
- 用户自导入UGC内容不断丰富

**第二层：斩题复活**
- 7天后题类/题目自动复活
- 微信订阅消息推送召回："你7天前斩掉的'英语虚拟语气'题类已复活"
- 复活后重新出现在刷题列表中，用户可重新挑战

**第三层：完成感驱动**
- 斩完一个题库 → 文字徽章标识
- 系统推荐："斩完数学的人都继续学了英语，你要来吗？"
- 首页从"刷题"变为"选择下一个征服目标"

### 5. 管理后台

使用**微信云开发CMS**，零代码搭建。功能包括：
- 题库批量导入（上传Excel → 解析入库）
- 题目增删改查
- 知识点管理
- 资讯/真题内容发布
- 用户反馈查看

### 6. 商业模式

| 收入来源 | 说明 |
|---------|------|
| Banner广告 | 首页内容区下方、答题结束页 |
| 激励视频广告 | 补签卡、额外题目额度 |
| 全部功能 | 免费 |

## 核心算法

### 智能选题算法

```
每题优先级 Score = w1×(1-masteryRate) + w2×difficultyMatch + w3×lastSeenDecay + w4×errorBoost

参数说明：
- masteryRate：该知识点用户正确率（0-1）
- difficultyMatch：匹配用户当前水平
- lastSeenDecay：距上次出现越久权重越高，e^(-days/7)
- errorBoost：错题权重提升1.5倍
- 默认权重：w1=0.35, w2=0.25, w3=0.25, w4=0.15

规则：
- 已斩知识点不出题
- 同一Session内不重复出题
- 按Score加权随机选取
```

### 斩题判定算法

**官方题库（按题类斩）：**
```
每次作答提交后（按题类追踪）：
1. 查找该 userId + questionClassId 的 classSlashProgress 记录
2. 更新该题类的累计 answered 和 correct 计数
3. 如果 answered ≥ 3 且 correct ≥ 2 且 status != 'slashed'：
   → 触发斩题按钮（浮动按钮显示在刷题页面）
   → 用户点击斩题按钮 → status='slashed', 记录slashedAt
   → 自动跳转到该知识点下的下一个未斩题类
4. 7天后复活检查：
   → slashedAt距今≥7天 → status恢复为未斩状态
   → 推送订阅消息通知用户
```

**自导入题库（按单题斩）：**
```
每次作答提交后（按单题追踪）：
1. 查找该 userId + questionId 的 questionSlashProgress 记录
2. 更新 recentCorrect[]（保留最近10条记录）
3. 计算正确率 = recentCorrect中true的数量 / 10
4. 如果正确率 ≥ 80% 且 status != 'slashed'：
   → 弹窗确认 → 用户确认 → status='slashed'
5. 7天后自动复活
```

### 错题间隔复习算法（SM-2变体）

```
间隔序列：[1, 2, 4, 7, 15, 30, 60]（天）

复习流程：
1. 每日获取 nextReviewAt ≤ now 的错题 → 加入今日复习队列
2. 用户作答：
   正确 → currentIntervalIndex++ → nextReviewAt = now + 间隔[currentIntervalIndex]天
         → 如果 consecutiveReviewCorrect ≥ 3 → status='mastered'，从待复习中移除
   错误 → currentIntervalIndex=0 → nextReviewAt = now + 1天
         → consecutiveReviewCorrect=0
3. 复习完成 → 更新统计
```

## 离线与缓存策略

### 缓存层级

| 层级 | 内容 | 过期策略 |
|------|------|----------|
| 内存 | 当前Session题目数据 | Session结束释放 |
| wx.setStorage | 最近3个题库缓存、用户信息、设置 | LRU淘汰，10MB上限 |
| 云数据库 | 全部数据 | 永久 |

### 离线能力

- 有网络：正常从云端获取，同步更新本地缓存
- 弱网：优先使用缓存题目
- 无网络：可继续做缓存中题目，答题记录暂存本地队列（最多500条），联网后批量同步

## 边界情况与异常处理

1. **斩题误操作**：二次确认弹窗 + 每日3次回滚机会
2. **题目被删除**：错题本中对应题显示"题目已更新"并自动移除
3. **异常快速作答（<2秒）**：判定无效，不计入连续正确，提示"请认真作答"
4. **答案争议**：用户可通过"举报"提交争议，争议题暂停计分进入审核
5. **网络中断**：已加载题目继续作答，记录进本地队列，联网后同步
6. **本地存储满**：提示清理缓存，自动清理最久未用的题库缓存
7. **多端登录**：每次登录从云端拉取最新进度，本地进度取最大值合并
8. **OCR超限**：每人每日OCR上限20次，超限提示"今日已达上限，建议使用Excel模板导入"
